Don't Repeat Yourself (DRY)

WHY?

  • Encourages you to think in terms of style patterns
  • Naming of groups encourages rational organization of design
  • Encourages optimization and generalization of selectors
Examples borrowed from : http://vanseodesign.com/css/dry-principles/

Group re-usable properties

Step 1 – Group styles that use the same styles but have different values

Example:
{ background-color: #fff; border-color: #ccc; }
{ background-color: #fff; border-color: #bbb; }

Name the groups logically

Step 2 – name the groups so that name makes sense and describes what the grouping does. Names in the grouping should be similar

Example:
.white-background
{ background-color: white; border-color: gray; }
.black-background
{ background-color: black; border-color: white; }

Repeat Selectors Not Groups

Step 3 - Add selectors to various groups

Example:
.translation, .entry .wp-caption, #full-article .entry img, .recent-comment .comment-text, .roundup h3, .post-header-sharing, #post-categories td.label, #post-archive roundup h3, .subscription-manager ol, .light-white-background
{ background-color: #fff; border-color: #ccc; }

textarea:focus, input:focus, #author-email-form .input-focus, #respond p input:focus, .wpfc7 input:focus, .medium-white-background
{ background-color: #fff; border-color: #bbb; }
