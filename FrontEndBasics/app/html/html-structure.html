The Basics

Requires a DOCTYPE

<!DOCTYPE html>

Every web page should have 3 elements

<html>

  • the root element

<head>

  • External file links (.js, .css) - .js can but should not go here
  • Page script inside of <script></script> - NO!!!!!
  • Page styles – Only one!!!!
  • Meta Data
    • Keywords (mostly useless) - <meta name="keywords" content="HTML, CSS, XML, XHTML, JavaScript">
    • Page title - <title></title>Appears in browser header or tab
    • Description - <meta name="description" content="Free Web tutorials on HTML and CSS">
    • Special instruction - <meta http-equiv="refresh" content="30">
    • Security - <meta http-equiv="Content-Security-Policy" content="default-src ‘self’; style-src 'self'; script-src ‘self’" />
    • Mobile - <meta name="viewport" content="initial-scale=1, maximum-scale=1, user-scalable=no, width=device-width">
    • Character Set - <meta charset> - Important to limit security vulnerabilities
  • Any external resource found in head is a blocking resource
  • CSS has to go in the head BUT try to keep it to one resource

<body>

  • Goal to keep markup machine readable but also it make contextually readable by people
    • <main>
    • <section>
    • <article>
    • <header>
    • <footer>
    • <aside>
    • <nav>

    All of these were created because they were used in all websites using class names on <div> elements
